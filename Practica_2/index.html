PRÁCTICA N2 - GIT, GITHUB Y HTML
  1. ¿Qué es una rama (branch) en Git y para qué sirve?
    Respuesta: Es una línea independiente de desarrollo que representa una versión del código. Sirve para trabajar en nuevas funcionalidades, correcciones o experimentos sin modificar el código principal, permitiendo que varios desarrolladores trabajen simultáneamente en el mismo proyecto sin interferir entre sí.
  2. ¿Cómo se crea y se cambia entre ramas en Git?
    Respuesta:
    • Crear: git branch nombre_rama
    • Cambiar: git checkout nombre_rama
    También se puede hacer en un solo paso con git checkout -b nombre_rama (crea y cambia a la rama al mismo tiempo). En versiones más recientes, también se usa git switch nombre_rama para cambiar y git switch -c nombre_rama para crear y cambiar.
  3. ¿Cuál es la diferencia entre la rama main y otras ramas?
    Respuesta: La rama main (anteriormente llamada master) es la rama principal y oficial del proyecto, donde se encuentra el código estable y listo para producción. Las otras ramas son líneas secundarias creadas para desarrollar tareas específicas; una vez que el trabajo en ellas está terminado y verificado, se integran de nuevo a main.
  4. ¿Cómo se combinan ramas en Git (merge)?
    Respuesta: Primero se debe cambiar a la rama que recibirá los cambios (generalmente main) con git checkout main. Luego se ejecuta git merge nombre_rama_a_integrar. Si no hay conflictos, Git realiza la fusión automáticamente; si hay conflictos, se deben editar los archivos afectados, resolver las discrepancias y luego confirmar los cambios con git add y git commit.
  5. ¿Qué problemas pueden ocurrir al trabajar con ramas (conflictos) y cómo se resuelven?
    Respuesta: Los conflictos surgen cuando dos ramas diferentes modifican la misma línea de un archivo o cuando un archivo se modifica en una rama y se elimina en otra. Para resolverlos: Git marca los archivos con conflictos, se abren y se editan para decidir qué código mantener, luego se ejecuta git add para marcar como resuelto y git commit para finalizar la fusión. También se pueden usar herramientas de resolución de conflictos para facilitar el proceso.
